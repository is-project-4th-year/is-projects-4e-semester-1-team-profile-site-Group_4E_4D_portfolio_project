# Lessons Learnt from the GitHub Crash Program

During the GitHub crash program, we learnt important concepts and practices that improved our project management and collaboration skills:

- **Milestones:** We learnt how to create milestones to organize and track the progress of different project phases.
- **Issues:** We understood how to open issues to document tasks, bugs, and feature requests, making our work more transparent.
- **Project Board:** We practiced using the GitHub project board to visually manage tasks using columns like *To Do*, *In Progress*, and *Done*.
- **Pull Request Merging:** We gained experience creating pull requests to review and merge code changes systematically.
- **Resolving Conflicts:** We learnt techniques to handle merge conflicts and ensure our main branch stays stable.
